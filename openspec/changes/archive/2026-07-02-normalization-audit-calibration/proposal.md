# normalization-audit-calibration

## Why

The first live sweep of the normalization re-audit passes (2026-07-02 ~01:40‚Äì02:00 UTC) proved the re-audit machinery works (5/6 fixture aliases corrected with clean canonical mints) and exposed four defect classes, each verified against production D1 during planning:

1. **Double-detail specialization ids.** The alias re-audit re-decided `'atlantic sockeye salmon fillets'` (already correctly mapped to `salmon fillets, skin-on::species-atlantic-sockeye`) as a NEW specialization with match = that 2-segment id and detail = `species-atlantic-sockeye`; `buildResolution`'s unguarded `${match}::${detail}` concatenation minted the 3-segment id `salmon fillets, skin-on::species-atlantic-sockeye::species-atlantic-sockeye`, now live in production ‚Äî with the alias re-pointed onto it, a structural edge onto the 2-segment parent, **and the 2-segment parent orphan-merged INTO the 3-segment child** (its `representative` points child-ward), so a naive "merge overflow into prefix" would be refused by `mergeIdentities`' cycle guard. The hardening change capped NOVEL canonical ids at 2 segments (`validateCanonicalId`) but specialization id construction has no segment guard anywhere.
2. **Edge audit over-drops.** In one tick the direction check kept `chili powder::type-ancho‚Üíchili powder` but dropped the structurally identical `rotel (original)::heat-mild‚Üírotel (original)`, `snacking pickles::form-chips‚Üísnacking pickles`, `serrano or jalape√±o peppers::form-diced‚Üíserrano or jalape√±o peppers` (all "distinct products; substituting changes the product"), dropped legitimate membership/containment edges (`sweet maui mango habanero sauce‚Üíhot sauces (various)`, `bulldog sauce‚Üíhot sauces (various)`, `jellied cranberry sauce‚Üíjellies and jams (various)`, `honey raisins‚Üíraisins`), and resolved the cardamom 2-cycle as `neither` ‚Äî deleting both edges including the true direction. Production holds **39 `edge_drop` log rows**; the prompt's "distinct products never satisfy each other / is-the-identical-product" framing is the systematic cause.
3. **Co-resolution rejection amnesia.** The SKU co-resolution pass re-proposes the same rejected pair ‚Äî production: `pecorino romano`/`parmesan` sharing SKU `0001111050195` ‚Äî every 5-minute tick forever (`job_health` shows `mergeRejected: 1` every tick), one wasted classifier call each.
4. **Punctuation-variant duplicate.** The alias audit minted `salmon fillets skin-on` as its own node while `salmon fillets, skin-on` existed at cosine ~0.98 (the confirm chose novel over a near-identical candidate). Production already self-healed this instance (the re-confirm pass merged it), but each occurrence costs a wasted mint plus a later healing LLM call.

Per the repo rule, production converges through the pipeline only: the observed defect rows are this change's acceptance fixtures, verified against production after deploy ‚Äî no manual data surgery.

## What Changes

- **Specialization segment guard** (`buildResolution`, `src/ingredient-normalize.ts`): a SPECIALIZATION pick whose chosen match already carries a detail (`match` contains `::`) is demoted to SAME with the match ‚Äî never concatenated into a ‚â•3-segment id ‚Äî with the demotion logged. Covers capture and the alias re-audit (both construct resolutions through `buildResolution`); the re-confirm pass never constructs ids.
- **Alias-audit no-op keep guard** (`src/ingredient-alias-audit.ts`): a re-decision that resolves to the standing mapping's survivor ‚Äî including a NOVEL whose validated canonical id resolves to the standing survivor, and a guard-demoted specialization ‚Äî is applied as a keep (re-commit + stamp), never as a fresh mint that only re-derives the standing id.
- **Segment-overflow repair reconcile** (deterministic sub-pass of the `ingredient-normalize` job): each tick, any surviving auto node whose id has ‚â•3 segments is repaired onto its 2-segment prefix ‚Äî plain merge when the prefix survives elsewhere, a **re-root** (prefix's `representative` cleared, overflow's pointed at the prefix) when the prefix currently resolves to the overflow node (the live production case), and a prefix mint (embedding NULL, backfilled) when no prefix node exists. Idempotent, logged, human nodes exempt.
- **Structural edge exemption + guarantee** (`src/ingredient-edge-audit.ts`): an auto edge `X::detail ‚Üí X` (from is exactly the to plus one detail segment, and from is a surviving node) is definitionally valid ‚Äî the audit keeps + stamps it deterministically with no model call and never deletes it, including as the reverse side of a 2-cycle. A deterministic per-tick pre-pass guarantees every surviving `base::detail` node has an edge to its base (born-stamped inserts; missing base nodes minted), restoring the wrongly-dropped rotel/pickles/diced class, and sweeps representative-resolved self-loop auto edges regardless of audit stamp (clears the post-repair salmon self-loop).
- **Direction-check recalibration** (`confirmSatisfiesDirection`, `src/ingredient-classify.ts`): satisfies redefined as "acceptably fulfills a request for" (not "is the identical product"); membership means the member fulfills a request for the category concept; whole‚Üíderived direction reinforced; few-shots added from the production mistakes (`honey raisins‚Üíraisins`, `sweet maui mango habanero sauce‚Üíhot sauces (various)`) alongside true-drop few-shots (`semolina flour‚Üíall-purpose flour`, `fruit pectin‚Üíjellies and jams (various)`, `frozen fruit mix‚Üídried fruit blend`).
- **One-shot edge-drop replay** (sub-pass of the `ingredient-edge-audit` job): every `ingredient_normalization_log` `edge_drop` row is re-evaluated once under the recalibrated validator ‚Äî endpoints parsed from the row's `'from -[kind]-> to'` term with a strict pattern ‚Äî and edges whose recalibrated verdict holds are re-inserted (born-stamped, logged `edge_restore`). A row whose reverse edge still stands is re-decided as a PAIR by that same single direction check (2-cycle semantics): a forward-only verdict restores the dropped edge and deletes the wrongly-kept reverse even when it carries an earlier keep stamp ‚Äî converging the production mislabel casualty (`whole frozen chicken`/`chicken tenderloin`) ‚Äî while human and structural standing reverses are deterministically immune. Rows are marked in their log detail (`replayed_at`); deterministic drops (self-loop, human-reverse), structural rows, merged-away endpoints, and unparseable terms are marked without a model call. Post-change `edge_drop` rows are born-marked, and edge log rows gain structured `from`/`to`/`kind` detail fields going forward, so the pass drains ~39 rows and quiesces.
- **Co-resolution rejection memory** (migration `0036_coresolution_rejections.sql` + `src/ingredient-normalize.ts`): a confirm-rejected pair is remembered in a small shared D1 table keyed by the pair's surviving ids (canonically ordered) with a decision time; remembered pairs are suppressed from re-proposal for a long backoff (30 days), retried once after it (a re-rejection refreshes the memory), and become eligible immediately when later merges change either survivor (the key changes). Survives restarts; suppression counted in the job summary.
- **Lexical-identity fast path + prompt rule** (defect D, in scope): a queued term (or re-audited variant) whose punctuation-insensitive lexical form exactly equals a known node id or alias variant resolves SAME deterministically with no model call (skipped on ambiguity ‚Äî two survivors sharing a form); the identity-confirm prompt additionally teaches that a term differing from a candidate only in punctuation, pluralization, or word order is SAME.

All passes stay bounded, idempotent, and self-quiescing; no new job names ‚Äî the sub-passes fold into the existing `ingredient-normalize`, `ingredient-alias-audit`, and `ingredient-edge-audit` jobs with additive-only `job_health` summary fields.

## Capabilities

### New Capabilities

(none ‚Äî all changes extend the existing ingredient-normalization capability)

### Modified Capabilities

- `ingredient-normalization`: ADDED requirements for the specialization segment guard, the segment-overflow repair reconcile, the structural edge guarantee, and the one-shot edge-drop replay; MODIFIED requirements "Canonical nodes and the full-id join" (ids capped at one detail segment), "Conservative collapse and prep-versus-product stripping" (lexical-identity fast path + punctuation prompt rule), "Stable ids with union-find merges" (co-resolution rejection memory), "Rolling re-audit of pre-hardening alias decisions" (no-op keep guard), and "Rolling re-audit of auto satisfies edges" (structural exemption + recalibrated direction semantics + born-marked drop rows).
