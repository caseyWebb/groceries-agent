## Why

The `recipes/` directory is empty, so the agent has no cookbook to plan menus from. Casey has a 63-recipe library exported from ReciMe (`recime-export.html`) that should become the seed corpus. The export is sparse — it carries titles, servings, prep/cook times, ingredients, and instructions, but none of the judgment fields the schema needs (protein, cuisine, style, dietary, etc.) and no source URLs — so a raw dump is not enough; the import needs a deterministic extraction followed by an enrichment pass.

## What Changes

- Add a kept-but-unwired importer script (`scripts/import-recime.mjs`) that parses the ReciMe HTML export into conformant `recipes/*.md` files.
- The importer drops non-recipe cards (the exporter's "All Your Recipes… ReciMe" landing card has zero ingredients/steps), exact-dedups by slug, and **merges** cookbook memberships into `tags` (a recipe filed in both `pastas` and `comfort-foods` keeps both tags rather than being discarded).
- Each extracted recipe is written with `status: active` (this is Casey's real library, not unvetted discoveries), `source: null`, deterministic fields populated (`servings`, `time_active`, `time_total`), and judgment fields left empty for enrichment. The importer refuses to overwrite existing files so re-runs never clobber enrichment.
- A dedicated naming pass assigns a clean `title` and plain slug across the whole corpus (drop SEO "Recipe" suffixes and marketing prefixes like "the best", resolve editorial headlines and foreign-name+gloss titles to the actual dish, preserve foreign dish names) — globally unique and stylistically consistent.
- An enrichment (beautify) pass fills the judgment frontmatter, curates `ingredients_key`, tidies recipe bodies (re-splitting any wall-of-text instructions), gut-checks the extracted times/servings (flagging — not silently fixing — anything implausible), and recovers `source` URLs **only** when the linked page's ingredients/steps actually match the recipe, not on a bare title hit.
- A final reconciliation pass wires `uses_components` / `produces_components`, reviews near-duplicates (e.g. two distinct `Pasta e Fagioli`, stovetop vs. pressure-cooker `Butter Chicken`) without auto-merging, and validates to zero warnings.
- Populate `_indexes/recipes.json` (and components) as a side effect of the existing build.

## Capabilities

### New Capabilities
- `recipe-import`: Extracting an external recipe export (ReciMe HTML) into conformant `recipes/*.md` files — covering non-recipe filtering, exact-slug dedup with tag merging, clean slug naming, deterministic field extraction, judgment-field enrichment, disciplined source recovery, and component reconciliation.

## Impact

- **`recipes/`**: populated from empty to 63 recipe markdown files.
- **`scripts/import-recime.mjs`**: new throwaway-but-kept Node script (uses the existing `gray-matter` / `js-yaml` deps so emitted frontmatter is byte-compatible with the validator); not wired into any agent flow.
- **`_indexes/recipes.json`, `_indexes/components.json`**: regenerated by `scripts/build-indexes.mjs`.
- **`recime-export.html`**: consumed as input, then **deleted** once the import completes (source data now lives in the recipe files). The exporter is the [ReciMe Recipe Exporter](https://chromewebstore.google.com/detail/recime-recipe-exporter/nbmmcjlploegpicloeoknlgdblcbmoga) Chrome extension (no source URLs present in its output).
- No changes to validation rules, MCP tools, or the Worker.
