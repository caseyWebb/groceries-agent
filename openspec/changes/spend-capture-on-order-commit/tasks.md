## 3. Send-record snapshot on the `place_order` flush

- [ ] 3.1 `src/spend.ts` (part 1 — snapshot): `SendSnapshot`/`SnapshotLine` types and `snapshotStatements(env, tenant, send, lines)` returning prepared statements (`order_sends` insert + `order_send_lines` inserts, insert-or-ignore) for composition into the advance batch; `deleteSendStatements(sendId)` for the rollback compensation.
- [ ] 3.2 `src/order-shapes.ts`: `PlaceOrderResult` gains `send: { recorded: boolean; id?: string; error?: string }` (leaf module, no workerd types).
- [ ] 3.3 `src/order.ts` `placeOrder`: widen the `advanceInCart` dep receipt to `{ inserted, sendId? }` and pass it to `rollbackInCart`; report `result.send` honestly (recorded with id on success; `{ recorded: false, error }` when the snapshot build failed but the flush proceeded — design D4). Preview/empty-resolve short-circuit unchanged (no send).
- [ ] 3.4 `src/order-tools.ts` `runPlaceOrder`: build the snapshot lines before the advance — per resolved line: prices from `ResolvedLine` (`unit_price` = promo when `isOnSale` else regular; `savings` via `deriveSavings`; `estimated: 0`), `department` via `departmentFor` (memo batch-read for the line keys; kind/domain from the stored row by key, plan-derived lines are food), `provenance` per design D6 (stored-list keys ∪ derived-plan keys ∪ non-empty `for_recipes` → `planned`; else `impulse`), `for_recipes` from the to-buy line. Thread stored-row kind/domain and the derived-need key set from the data already in hand (no extra reads). `makeAdvanceInCart`/`makeRollbackInCart` compose the snapshot/delete statements and the `sent_in` stamps into the existing batches (`advanceInCartRows`/`rollbackInCartRows` in `src/session-db.ts` gain an optional `sendId`). A snapshot-build throw degrades to a bare advance + `send.error` — it must never fail the flush.
- [ ] 3.5 `place_order` tool description (`src/order-tools.ts`): state the send-record side effect, the `send` result field, and that the snapshot prices are send-time quotes (the existing SKU-not-price guarantee text extends naturally).
- [ ] 3.6 Tests (`test/order.test.ts` / order-tools suites): snapshot rides the advance batch (send exists iff advance succeeded); rollback deletes it; preview writes nothing; provenance mapping across the four line origins (design D6 scenario); department stamping incl. a household row and a cold id; `send` reported honestly on a forced snapshot failure.
